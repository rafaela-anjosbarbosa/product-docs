# Product Docs

Bem-vinda(o) ao starter de **Docs-as-Code**.

O objetivo é organizar a documentação **funcional e comportamental** do sistema (telas, componentes, requisitos, regras e fluxos) de forma:
- versionável em Git
- rastreável
- fácil de publicar (MkDocs)
- pronta para automação e IA

Use o **Sistema X** como exemplo e replique o padrão para outros sistemas.
